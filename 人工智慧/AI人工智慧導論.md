### 認識AI

1. 如何做人工智慧
   - 培養觀察力。
   - 訓練問問題。
   - 聊天機器人用遞歸神經網路納入前一次狀態。
2. 函數f是解答本
   - 定義域X：所有可能問題的集合。
   - 值域Y：所有可能答案的集合。
3. 如何使用AI解決問題
   - (1) 提問。
   - (2) 將問題轉換成函數。
   - (3) 收集歷史資料，注意過度擬合。70%訓練資料、30%測試資料。
   - (4) 以機器學習、神經網路建構函數學習機。
   - (5) 學習或訓練機器學習機，建模，注意損失函數。
4. 近期AI大爆發主因及挑戰
   - 複雜的軟體、計算能力提升、大數據累積。
   - 電腦無法提出好問題，且無法自動建造函數學習機。
5. 常見與機器學習有關的演算法
   - 分類：對應監督式學習，將未知新訊息歸納到已知資訊類別，根據多個特徵將分類結果加上標籤，如生物學分類。
   - 分群：對應非監督式學習，資料集沒有明確分類，需以特徵區分成不同群集。
   - 一般表示方法：2~3個維度用線性分類器，超平面n維度用數學方程式。
6. 監督式學習
   - 支持向量機 (SVM)：根據核函數決定以線性或非線性分類器分群，找出最大支持向量(兩分類間距離分類線最近的點取最大距離)
   - 決策樹 (decision tree)：建立多元樹，包括根節點、節點、葉，透過迭代試驗找出一分類原則有較高正確率的資訊獲利。
   - K近鄰 (KNN)：k值表示比對k個已分類最接近的資料點，通常為奇數。
7. 非監督式學習
   - K平均 (k-means)：隨機分成k個群集，持續迭代計算出每個分群的重心與重新分群。

### 神經網絡架構

1. 標準NN = 全連結神經網絡
   - 使用時首先決定輸入、輸出的維度。
   - 人工神經網絡：輸入層、隱藏層、輸出層。連接神經網絡、前饋神經網絡。
   - 決定隱藏層數量，以及每個隱藏層的神經元數量。隱藏層數量大於3層稱為深度學習。
   - 損失函數 L 的變數有二：(1)權重數值 wi 越大，輸入的越重要。(2)偏值 bi 用來調整總刺激。
   - 激活函數判斷刺激夠大為1，可忽略者0。
   - 學習速率 r
   - 梯度下降法/反向傳播法：梯度 gradient L 與損失函數 L 方向相反。
2. 卷積CNN：常用於圖形辨識
   - 卷積層：機器以多個過濾器查看照片，分次檢查不同特徵，將特徵強度分數記錄在計分板。
   - 池化層：從固定選區大小(如2*2)挑出指定值(最大、最小、平均)。
   - input-卷積層-池化層-卷積層-池化層-...-全連結層-output
3. 遞歸RNN：保有前次輸入記憶
   - 隱藏狀態是輸出會當作下次的輸入，也會同時傳給同一層鄰居。
   - 適用於與時間有關的判斷，和需要參考前後文的判斷。如翻譯機、接續創作文章或圖片。

### 圖像辨識

- 圖像是由色塊/像素組成的矩陣，解析度是像素的行數列數。
- 空間濾波器：對影像的預處理程序，包括調整大小、旋轉，透過卷積運算(移動 stride->對齊->計算乘積和->補零)提高精確度。

1. 濾波器(filter)/遮罩(mask)/核心(kernel)，如3*3方形。
   - 平均濾波器(average filter)/平滑濾波器(smoothing filter)、權重平均濾波器(weighted average filter)：取出卷積結果，減少雜訊，模糊化。
   - 中值濾波器(median filter)：取出中間值，濾除雜訊。
   - 索伯濾波器(Sobel filter)：透過離散性差分(discrete difference)計算圖像亮度之梯度(gradient)，分成Gx和Gy濾波器，用於邊緣檢測。
2. 深度神經網絡(DNN)：輸入層 + n層隱藏層 + 輸出層
3. 卷積神經網絡(CNN)：簡化DNN完成訓練，以AlexNet為例(2012年ILSVRC挑戰賽)。
   - 卷積層(convolutional layer)：利用卷積核概念，接收特徵圖(feature map)，提取特徵產生新圖。
   - 池化層(pooling layer)：找出局部特殊值，均分成等大區域從該區中取出最大值或平均值，降低圖資複雜度。
   - 全連接層(fully connected layer)：利用卷積運算，將兩不同位置之特徵向量進行轉換，交付分類器進行分類。
   - 激活層(activation layer)：解決線性系統的疊加問題，引入非線性函數運算到每次的線性運算中。包括 sigmoid 函數、tanh 雙曲正切函數、ReLU 線性整流函數(最常使用)。
   - 輸出層(output layer)/標準化指數層(softmax layer)：將計算結果定於0~1之間，前面會連接全連接層，後面結果即為分類器。
   > AlexNet：輸入層 -> 卷積層1 -> ReLU激活層 -> LRN(Local Response Normalization) -> 池化層 -> 卷積層2 -> ReLU激活層 -> LRN -> 池化層 -> 卷積層3 -> ReLU激活層 -> 卷積層4 -> ReLU激活層 -> 卷積層5 -> ReLU激活層 -> 池化層 -> 全連接層1 -> ReLU激活層 -> Dropout -> 全連接層2 -> ReLU激活層 -> Dropout -> 輸出層(Softmax, 1000類)
4. 訓練模型：
   - (1)資料集：訓練集、驗證集、測試集。
   - (2)解決過度擬合：增加資料量(=資料增強)，訓練時隨機停止某些神經元(=加入丟棄機制 dropout，降低深度學習複雜度)。
5. 圖像辨識應用
   - 自動化整理相簿
   - 建立視覺資料之圖片網、影片網
   - 發展家庭保全系統，影格率/fps。
   - 使用用戶生成內容/用戶側寫，完成互動式行銷，提高廣告轉化率。
6. 免費數據庫
   - [ImageNet](https://www.image-net.org/)
7. 圖像辨識研究：陽明交大人工智慧與多媒體實驗室，史丹佛大學電腦視覺研究室，美國麻省理工學院電腦科學與人工智慧實驗室，訊連科技，玩美移動、谷歌深度大腦。
8. 未來發展：資料策略，資料長 CDO。

### 影像辨識

1. 影像是n個圖像組成，連續圖像，在二維圖像加上時間維度。每秒取樣幾個影格/楨(FPS)。前處理：資料量大，取樣頻度，再取樣率高低。
2. 動作估計(motion estimation)：提取影像中重要資訊的方法，將成對的水平與垂直位移量動作向量v建立估測結果，計算動作向量(motion vector)，能用於影像識別，也常用於影像壓縮。
   > 假設有三，物體於相鄰兩影格間位移量不大，物體不隨時間變色，物體不隨時間形變。物件層級大於區塊層級，製作動作向量圖設時間點第t影格、第t+1影格，分割畫面 m x n 個的 d x d 大小區塊，計算絕對誤差值總和。<br>
   > RGB 影像轉換成動作向量時，每個區塊 d = 8 px，動作向量圖的資料量是原本的 1/(8 * 8) * 2/3。1/(8*8) 為空間壓縮：每 8×8 像素區塊以一個動作向量代替；2/3 為通道壓縮：RGB 有 R、G、B 三個通道，動作向量只有水平位移 Vx 與垂直位移 Vy 兩個值，合計資料量約為原始的 1/96。
3. 物體追蹤(object tracking)：以深度神經網絡的滑動視窗(滑窗/sliding window)掃瞄圖像，找出圖像中特殊表示並輸出類別，計算各組特徵與待追蹤目標物體與第t影格特徵間的最小差異值，當小於某個閾值/臨界值/門檻時可得知追蹤物體的新位置，建立移動軌跡。
4. 行為辨識(action recognition)：實作時有四個難題要解，受取像、方位、光線、角度影響大，相同物種的同一類動作變異大，取樣率不同會被誤判成相同動作，單一圖像有許多干擾物體存在。第三點解法是再採樣策略。第四點解法是搭配注意力機制可濾除不必要的資訊。
   - 再採樣策略：單影格行為識別、多影格延遲混合行為識別、多影格早期混合行為識別、多影格緩慢混合行為識別。
   - 注意力機制(attention mechanism)：使用注意力遮罩除掉不必要物體，兩個相同張量的原始圖像與注意力遮罩相乘結果。

### 語音辨識

1. 基本概念
   - 音訊(audio signals)：介於20 Hz到20k Hz的訊號。
   - 音訊參數：(1)取樣率(sample rate)分成電話8k、語音識別16k、CD錄音44.1k Hz。(2)位元解析度(bit ewsolution)。(3)聲道數(channels)分成單聲道、雙聲道/立體聲。
   - 提取聲紋特徵時，在音框裡必須包括多個基本週期，以數學或向量表示。每秒出現的音框數，稱作音框率(frame rate)。
   - 音訊特徵包括3項：音量/強度(intensity，單位 dB)、音高/頻率(單位 semitone)、音色(用FFT或MFCC轉換後變得平滑)。

2. 語音辨識
   - 根據使用者，分成語者相關(speaker dependent)、語者獨立(speaker independent)。
   - 根據功能難度區分，語音命令、關鍵詞偵測、聽寫、對話。
   - 提取音訊特徵：用MFCC抽出13、26、39維度的向量。
   - 未來：製造業設備異音判定，災難安全預警，遠距照護，醫療儀器或病患言語偵測。
   
   - 最簡單的是「語者相關」+「語音命令」。如Sony Ericsson T18拉麵廣告。
      - 錄製多次命令語音，進行端點偵測與計算MFCC。
      - 採用動態時間扭曲(dynamic time wrapping, DTW)，能根據講話音色、語音速度比對，達到對位(alignment)，找出最短的距離。
   - 最複雜的是「語者無關」+「對話」。如Siri、Alexa、Coogle Home。
      - 語音特徵仍用MFCC取出，但用不同的聲學模型表示子音、母音，用MLE或GMM或HMM計算出機率密度函數(PDF)。
      - 音素(phoeme)是指人類語音中能區別不同發音的最小聲音單位。如ㄚ是a一個，ㄞ是ai兩個。
         - 單音素建模：p, i, ng, a, n
         - 雙連音素建模：運用右文相關，sil+p, p+i, i+ng, ng+a, a+n, m+sil
         - 三連音素建模：運用左右文相關。
      - 資料建模：用GMM或DNN轉成拼音，轉成音素，再轉成以雙連音素建模，最後串接成HMM模型。
      - 資料推論：後續使用這個模型，以Viterbi search計算最大似然率，即為比對結果的符合度。必要時用語言模型等的詞格搜尋，在有限時間內回傳識別結果。

   - 哼歌選唱(query by singing/humming, QBSH)的辨識步驟
      1. 對輸入音訊進行音高追縱。
         - 為了產生時間對音高向量的資料，切音框、計算自相關函數(ACF)結果、計算音高。
      2. 根據音高向量，比對資料庫的歌曲。主要是看母音造成喉嚨規律震動。
         - 歌唱速度差異，可用線性伸縮(linear scaling)調整，找出壓縮或拉長的哪個倍數較為匹配。
         - 複雜音訊音樂(polyphonic audio music)包括人聲和背景音樂，難以計算出人聲的音高。==>主旋律擷取(audio melody extraction)。
         - 隨著資料庫歌曲數提升，計算量也會增加。
      

### 自然語言處理

1. 基本概念
   - 自然語言(natural language)，等於人類語言。
   - 資料庫的檢索架構：倒置列表(inverted list)的雜湊表。
      - table 有兩欄：詞彙(關鍵詞)、起始鏈結。
      - 資料庫行為：使用者提出一個問句，擷取此問題的關鍵詞，找出最適合的一組關鍵詞，查詢此雜湊表並計算雅卡爾係數。
   - 導入深度學習，建立詞向量檢索，維度通常是300維。
      - 建立詞向量(建立詞的數值化)、特徵圖(feature map)，包括單詞組、雙連詞組、三連詞組，透過最大池化都各取一組串聯，產生句子表示(sentence representation)。
      - 把使用者問題句，投射到這個向量空間，在比較曾計算餘弦相似度，產生答案推薦排序。
   
2. 常見手段
   1. 斷詞處理：界定單一詞彙的邊界，包括未知詞的處理。如中研院斷詞系統、史丹佛大學的自然語言軟體(Standard CoreNLP)。
   2. 詞性標記：根據上下文標定每個詞彙的詞性(part of speech, POS)，並把相同的叢聚一起。
   3. 詞意消歧：根據上下文辨識句子中每個詞的意思。如「關門」有打烊或停業的兩種意思。
   4. 語法剖析：分析句子結構，產生剖析樹(parse tree)與相依剖析(dependency parsing)。

3. 問答系統的答案選擇模型
   - A. 詞袋表示法+雅卡爾係數
      - 雅卡爾係數(Jaccard coeefficient)計算兩個詞袋(問題-參考答句、問題-參考問句)的相似度。<br>
         - J = (A, B) = A 與 B 取交集 / A 與 B 取聯集<br>
      - 詞袋(bag of words)表示法：僅對目標問句做斷詞處理，擷取關鍵詞，放到 A。<br>
      
      > 解決比對 B 參考答句的長句分母大的問題？ => 把答句拆成多個「句子」，分開計算雅卡爾係數。<br>
      > 解決比對 B 參考答句上下文的代名詞？ => 代名詞消解(pronoun resolution)。
      > 若問題-參考答句、問題-參考問句的相似度過低，即雅卡爾係數不高，理應不推薦參考答句 (= 不答覆)。

   - B. 詞向量+餘弦相似度
      - 詞向量(word vector)：把每個詞彙都用一個向量標示，用不同尺寸過濾器進行卷積運算。
      - 餘弦相似度(cosine similariy)計算兩個詞彙之間的關聯程度。
