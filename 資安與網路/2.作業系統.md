## 2-3. Linux伺服器端與用戶端
- 伺服器、服務、埠、用戶
    - A server is said to be “listening” (監聽) on a port when it has associated itself to that port.
    - A port is a reserved network resource used by a service. 
    - 常用port配置 (wiki: [TCP/UDP埠列表](https://zh.wikipedia.org/wiki/TCP/UDP%E7%AB%AF%E5%8F%A3%E5%88%97%E8%A1%A8))
    - well-known ports: FTP 20/21, SSH 22, Telnet 23, SMTP 25, DNS 53, DHCP 67/68, TFTP 69, HTTP 80, POP3 110, NTP 123, IMAP 143, SNMP 161/162, HTTPS 443. 
    - Clients (= client applications) are programs or applications designed to communicate with a specific type of server.

- 伺服器管理
    - 原理：用設定檔(configuration file)管理服務，在服務啟動時會將此檔載入記憶體，根據設定項調整。若有修改要重啟服務才會生效。此文件內常見設定項有：埠號(port number)、主機來源位置(location of the hosted resources)、用戶授權細項(client authorization details)。
        - 查看設定檔：``` cat /etc/*.conf ```
        - 格式不固定，常以option = value、ipvar option value為主，hash (#)是註解。
        - 範例：Nginx (a lightweight web server for Linux)
    - 裝置強化(hardening devices)
        - 目標：實作裝置安全，保護管理權限。
        - 項目：確認實體安全。減少安裝套件。停用未用服務。使用SSH並停用root帳戶。保持系統最新更新。停用USB自動偵測。使用強密碼組合。定期更新密碼。避免使用舊密碼。

- 監測日誌
    - 種類：Application logs, Event logs, Service logs, System logs，包括對後台常駐程式(daemon)的監控。
    - 位置：/var/log/*

- 檔案系統
    - 種類：ext2 (default file system), ext3 (journaled file system, max file size = 32TB), ext4, NFS, CDFS, swap file system, HFS Plus, APFS in Apple devices, MBR.
    - mounting是指派目錄路徑(directory)到對應位置(partition)。mounting point是此目錄路徑，相當於Windows哪一槽別(drive letter)的概念。
    - 查目錄(directory)用 `mount` 指令。
        - 根目錄是"/"，如 `/dev/sda1 on / type ext4 (rw,relatime)`，the root file system was formatted as ext4 and occupies the first partition of the first drive (/dev/sda1)。
    - 查檔案權限(file permissions)用 `ls -l` 指令。
        | 第1碼 | 第2~10碼 | 第二欄 | 第三四欄 | 第五欄 | 第六七八欄 | 第八欄 |
        | -- | -- | -- | -- | -- | -- | -- |
        | -是file，d是dir | read/write/execute for user, group, others | 2=hard link, symbolic link | 檔案擁有的user+group | bytes | 最後修改時刻 | 檔名 |
        - 建立強連結(hard link)檔案用 `ln space.txt space.hard.txt` 指令。原檔會隨新檔增加或減少內容，但不會因新檔rm移除而消失。只能在同一個磁區 (Partition)中建立，也只能連結檔案，不能連結目錄。inode是在檔案系統中相同連結位址建立不同檔名。
        - 建立軟連結(symbolic/soft link, symlinks)檔案用 `ln -s test.txt mytest.txt` 指令。類似MS-Windows下的「建立捷徑」的功能，主要用於簡化路徑。直接連上原檔路徑，故原檔被刪除時，新檔也會讀取錯誤。可用於跨檔案系統(=磁區)的連結，以及目錄間的連結。
        - octal values: 定義檔案權限rwx，換算成八進位。如rwx視為7。
        - 複寫檔案權限(override file permissions)僅限於root user。

- GUI介面
    - Gnome, KDE。Ubuntu Linux使用Gnome 3。
    - 畫面組成：Apps Menu, Ubuntu Dock, Top Bar, Calendar and System Message Tray, Activities, Status Menu.

- 執行Linux Host
    - 安裝時，使用package manager安裝套件。
    - 檢查及更新套件版本用 `apt-get update`、`apt-get upgrade`。
    - process是程式的執行實體，會有單獨process ID，會以 `ps`、`top`、`kill` 管理。
    - 多工作業系統(multitasking operating system)：forking是kernel允許建立多個process，caller process作為父流程，待子流程建立後兩者互為獨立，有兩個PID同時執行相同程式碼。如Apache可同時處理多個request。
    - 隱匿程式(rootkit)：遭感染電腦自動安裝或駭客手動安裝，隱藏入侵行為、移除安裝痕跡、竄改偵錯和修正工具，變更OS執行原碼。
        - 檢查方式包括behavioral-based methods, signature scanning, difference scanning, and memory dump analysis。booting the computer from trusted media。
        - 移除方式包括硬體更換(hardware replacement for firmware rootkits)、重新安裝作業系統(re-installation)。
        - chkrootkit 無法完全檢查出問題位置或差異(discrepancies)，但實為不易追查行蹤和所在位置。

## 2-4. Windows作業系統

- 儲存硬體裝置
    - 近前：硬碟(hard drives)、光碟機(optical drives)、快閃記憶體(flash storage)
    - 早期：穿孔卡(punch cards)、紙帶(paper tape)、錄音帶(magnetic tape)、錄音盒(audio cassettes)
    - MS收購DOS (Disk Operating System)，開發出MS-DOS，提供檔案系統(file system)的讀取和寫入。自從1993年起，超過20個Windows版本(release)上線。
        - Windows 1.0是1985年推出，當時已有GUI在MS-DOS系統上執行，單一process (single-process, single-user MS-DOS)。
        - Windows XP：從這個版本開始支援64位元OS (64-bit edition)，可使用大型資料庫、科學計算、高清晰度數位影像。64位元的電腦與作業系統可相容並執行較舊的32位元程式，但64位元程式無法在較舊的32位元硬體上運行。
        - Windows 10是建立在Windows NT (Windows New Technologies)基礎上，亦即OS搭配多個使用者的process。目前仍可透過Window Search調用出cmd。
        - MS使用Windows File 
- Windows架構和作業
    - CPU執行時分成兩種模式
        - Kernel模式 (code執行報錯，會導致整個OS崩潰)
            - 硬體(hardware)
            - 硬體抽象層(hardware abstraction layer, HAL)：溝通硬體與作業系統核心、Kernel模式驅動。
            - 作業系統核心(OS Kernal)：與User模式的Window API互動。
            - Kenrel-mode Drivers：與User模式的Window API互動。
            - 檔案系統驅動：與User模式的Window API互動。
        - User模式 (code運行時，受限於kernel指定的儲存位址)
            - Windows API：與服務(applications)溝通，與Kernel模式的作業系統核心、檔案系統驅動互動。
            - User-mode Drivers：與Kenrel-mode Drivers、Window API互動。
    - Windows檔案系統-儲存
        | 檔案系統 | 主要平台 | 支援度 | 優點 | 缺點 |
        | ---------- | --- | --- | --- | --- |
        | **exFAT**  | Windows、macOS、Linux | 高 | 輕量、限制少、適合隨身碟 | 不支援檔案權限、安全性較低 |
        | **HFS+** (已被APFS取代) | macOS | 中 | 支援大型檔案與名稱 | Windows不支援、較舊 |
        | **EXT** | Linux | 中 | 穩定、安全、常用於伺服器 | Windows不原生支援  |
        | **NTFS** | Windows | 高 | 安全性好(安全描述元(Security Descriptors))、可靠、多功能、支援大型容量、支援檔案系統層級加密、記錄多種時間戳(MACE=修改,存取,建立,目錄項目修改) | macOS寫入需額外軟體 |
        - NTFS格式化
            - Partition Boot Sector	前16磁區，包含MFT的位置。最後16磁區為備份。
            - Master File Table (MFT) 記錄分割區內所有檔案與資料夾的位置、屬性、安全資訊等。
            - System Files 隱藏檔，用於管理 NTFS 結構。
            - File Area 真正存放檔案與資料夾的區域。
        - 在NTFS可另訂資料串流Alternate Data Streams (ADS)，以原檔名後接:ADS，透過`C:\ADS> dir /r` 可查到這個隱藏資料存放方式。
            - 資安疑慮：被用於隱藏檔案及內容，可以做為將惡意程式寄宿於一般檔案上，或者是作為資料洩漏、資料竊取的工具。
    - Windows開機流程(Windows Boot Process)
        - BIOS (Basic Input-Output System)：1980年代，古老、用 MBR 開機。
        - UEFI (Unified Extensible Firmware Interface)：現代化，更安全。
        1. 執行Bootmgr.exe：切換模式到受保護模式(protected mode)使全部記憶體可使用，讀取BCD判斷從休眠恢復或冷開機。
            - BIOS：MBR、POST兩步驟，才會執行執行Bootmgr.exe
            - UEFI：用 ESP + EFI 檔案開機，執行執行Bootmgr.exe
        2. 根據BCD判斷要執行從休眠恢復的Winresume，還是從冷開機的Winload
            - 執行Winresume.exe，讀取 Hiberfil.sys 以恢復休眠前的系統狀態，恢復Winlogon。
            - 執行Winload.exe，建立硬體配置紀錄於Registry，使用KMCS (Kernel Mode Code Signing)驗證驅動程式的數位簽章。
        3. 載入核心，驗證並驅動Ntoskrnl.exe，設定HAL（硬體抽象層）。
        4. SMSS (Session Manager Subsystem)啟動系統與使用者環境，包括讀取Registry，啟動Winlogon，準備桌面環境。
            - 自動啟用的服務和應用：登錄在registry的HKEY_LOCAL_MACHINE、HKEY_CURRENT_USER。
            - Msconfig.exe是在System Configuration設定。
    - Windows關機順序(Windows Shutdown)
        - 先關閉使用者模式(User Mode)應用程式，如果沒回應系統會提示你等待或強制關閉。再關閉核心模式(Kernel Mode)進程，如果無回應系統可能會卡住。
            - Shutdown 關機： 完全關閉電腦，電源停止供應。
            - Restart 重新啟動：用於安裝更新或解決問題。
            - Hibernate 休眠：將目前的開啟程式、檔案、記憶體內容寫入Hiberfil.sys，供下次開機恢復使用， 不耗電。

    - Windows應用程式
        - 應用程式(application)由一或多個process組成。
        - process是任何程式正在執行中，由一或多個執行緒(thread)組成。
        - processor能計算執行緒，也與同一時間電腦可執行的執行緒總數有關。
        - 查詢與設定在：Task Manager。
        - 當前process的執行緒無法被另一個process存取，因為是不同位址空間(address space)。
    - Windows服務
        - 特指支持OS和應用程式的背景程式process。
        - 查詢與設定在：Windows Services。(開機自動或手動啟用/停用)

    - Windows記憶體
        - 電腦在執行程式時，會把指令存放在RAM中，CPU則透過虛擬位址(virtual address)來存取記憶體。32位元電腦單一程序最大4GB，64位元電腦是8TB。
        - 虛擬位址不是記憶體的實體位置，以頁面表(page table)轉換成真正的實體位址(physical address)。
        - 當使用者模式程序需要存取核心(kernel)，如檔案、驅動程式、系統物件時，必須透過句柄(Handle) 來間接存取，不能直接存取核心記憶體（安全性原因）。
        - 管理工具：MS RAMMap，可查記憶體當前分配的核心、程序、驅動程式、應用程式。

    - Windows Registry
        - 階層式資料庫，以key和subkey組成，可達512層。
            - key有3種形式：REG_BINARY、REG_DWORD、REG_SZ。
        - Computer下方只有5個Registry hive
            - HKEY_CLASSES_ROOT：儲存Windows可辨識檔案類型列表，以及相關的程式。OLE註冊。
            - HKEY_CURRENT_USER：當前登入的使用者設定資訊。
            - HKEY_USERS：主機所有使用者資訊。
            - HKEY_LOCAL_MACHINE：主機系統軟硬體資訊。
            - HKEY_CURRENT_CONFIG：當前硬體組態列表。
        - 操作重點
            - 使用regedit.exe可以修改。(必須非常留意!!)
            - 若遭惡意程式新增啟動鍵值，可能在開機時隱藏執行，例如鍵盤側錄器(keylogger)，將嚴重危害系統安全。 -> 檢查所有開機啟動位置
            - 記錄使用者日常操作，包括曾連接的硬體裝置（含名稱、製造商、序號）與開啟過的文件、位置及存取時間。 -> 數位鑑識

    - Windows設定與執行
        - 使用帳戶種類
            - 系統管理員：預設關閉。Run as Administrator
            - 訪客(guest)：預設關閉。無須密碼
            - 本機使用者(local user)
                - 可被加入一至多個群組(group)。當權限彼此重疊時，某些權限如explicitly deny，會覆蓋其他群組的權限。以lusrmgr.msc進行管理。
                - 透過網域(domain)設定權限。使用者和每台電腦都必須向domain collector (DC)進行驗證，並覆蓋本機電腦或使用者帳戶的預設設定。
        - Windows CLI
            - 以cmd.exe開啟CLI，可瀏覽檔案目錄、執行bat檔案(不分大小寫)。可選參數(switch)用正斜線區隔。按壓Tab鍵自動填補指令。切換儲存裝置(storage devices)，只要用disk名稱加上:，如`D:`。
            - `net` 支援許多子命令，用作管理及維護OS使用。
        - Windows PowerShell
            - Cmdlets：這些命令會執行某個動作，並將輸出或物件傳給下一個要執行的命令。
            - PowerShell腳本：副檔名 .ps1 檔案，內含要執行的 PowerShell 命令。
            - PowerShell函數：這些是程式碼片段，可在腳本中被引用使用。
        - Windows Management Instrumentation (WMI)
            - 連線並管理遠端電腦，取得電腦元件資訊、硬體與軟體統計資料
            - 電腦管理(Computer Management) > 服務與應用程式 > WMI Control > 內容。
            - 禁止Winmgmt，或以防火牆等方式封鎖WMI，避免駭客透過WMI命令未留痕跡地連線到遠端電腦。
        - 工作管理員(Task Manager)
            - Processes 程序：正在執行的應用程式和背景程序，對應硬體、網路使用情況。
            - Performance 效能：提供硬體資源的使用率和歷史趨勢。
            - App history 應用程式歷史記錄：顯示Microsoft Store應用程式的資源使用歷史。
            - Startup 啟動：管理開機自動啟動的應用程式，並可啟用或停用。
            - Users 使用者：目前登入的使用者帳戶及其資源使用情況。
            - Details 詳細資料：提供PID、記憶體使用量，可結束或管理程序。
            - Services 服務：列出系統服務及其狀態，可啟動或停止服務。
        - 資源監視器(Resource Monitor)
            - CPU：CPU使用率、各程序的CPU消耗，以及關聯的服務和線程。
            - Memory：實體記憶體使用狀況，包括每個進程使用和可用記憶體。
            - Disk：磁碟活動，包括每個進程的讀寫操作、磁碟隊列和 I/O 數據。
            - Network：網路流量，包括各進程的網路使用情況、TCP連線和網路介面統計。
            - Overview：整合CPU、記憶體、磁碟與網路的使用狀況。
        - 事件檢視器(Event Viewer)
            - 紀錄分成Windows Logs、Application and Services Logs。
            - 事件分級information, warning, error, critical。
        - 檢查及設定網路
            - 網路與共用中心(Network and Sharing Center)
            - `nslookup cisco.com` 查詢DNS位址、紀錄。
            - `netstat` 顯示電腦的網路連線狀態。偵測惡意連線或可疑外部連線。
                - 以管理員身分執行cmd，輸入 `netstat -abno` 檢查任何可疑程式監聽輸入流量，再透過Task Manager把PID移除掉。
        - 存取網路資源
            - 使用Universal Naming Convention (UNC)
                - 根據IBM的SMB協議(Server Message Block)，開發了UNC連線。
                - 直接在Windows檔案總管輸入共享資源的UNC，路徑如`\\servername\sharename\file`，存取遠端共享資料夾。
                - 只限管理者能透過 `\\遠端電腦名稱\C$` 及 `admin$` 或 `print$`，進入由Windows自動建立的隱藏共享資料夾，遠端更新軟體或備份資料。
            - 使用遠端桌面協議(RDP)
                - 開啟遠端桌面連線，登入遠端主機。
                - 容易成為攻擊目標，建議以Zero Trust限制對內部主機的存取。
        - Windows Server
            - Windows Server 2000：雖然叫 Server，但技術上接近桌面版 Windows 2000（NT 5.0），屬於「客戶端型」延伸。
            - Windows Server 2003：正式的伺服器版本，基於 NT 5.2，開啟新一代 Windows Server 家族。
        - Windows Update Management
            - 服務包(service packs)、修補程式(patches)。
            - 自動偵測是否有最新更新。
        - Local Security Policy
            - 用於非網域的獨立電腦(stand-alone PC)，管理不在Active Directory domain的電腦。
            - 如何啟用：搜尋 Local Security Policy 並點擊程式。
            - 設定密碼(Password Policy)、帳號鎖定(Account Lockout Policy)、使用者權限(User Rights)、防火牆(Firewall Rules)及程式執行限制，確保電腦在單機或網路環境下都能維持安全。
        - Windows Defender與Windows Defender Firewall
            - restrictive security policy
            - permissive security policy
